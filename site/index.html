// About

Cold starts are the bottleneck

The problem

Before an AI model can respond to a single request, it has to load tens to hundreds of gigabytes of data onto GPU hardware. For Llama-3-70B, one of the most popular open models, this takes over 9 minutes. Every restart, every scale-up, every new server — the same wait. Expensive GPUs sit idle while data trickles in.

The solution

thaw snapshots the fully loaded state of a model and restores it in seconds. Think of it like hibernate on a computer — instead of booting from scratch every time, you pick up exactly where you left off. The restored model produces bit-identical outputs to the original, verified by automated testing.

Serverless inference

GPU endpoints that start in seconds, not minutes. Serve requests on demand without keeping expensive hardware running 24/7.

Cost reduction

Every minute of cold start is a minute of wasted GPU time. Faster starts mean more of your compute budget goes to actual inference.

Agent infrastructure

Clone a running AI session to explore multiple paths simultaneously. Fork parallel conversations from a shared starting point.

// Capabilities

Built for production inference

Pipelined DMA

Double-buffered CUDA streams with O_DIRECT. Overlaps disk reads with PCIe transfers. Rust+CUDA hot path hits 14.79 GB/s on PCIe Gen5.

Multi-GPU Tensor Parallel

Per-rank snapshots with automatic sharding. Freeze and restore across any number of GPUs. 17.2× on 2× A100 SXM with Llama-3-70B.

KV Cache Snapshots

Nobody else does this. Freeze prefix-cached KV blocks, restore with hash mappings intact. New requests skip prefill entirely.

Agent Forking

Clone a running AI session. Snapshot weights + KV cache, restore in a new process, fork parallel completions from shared context.

Native vLLM Integration

load_format="thaw" works with vLLM's official ModelLoader plugin system. Two-line API. pip installable.

Bit-Identical Output

Every benchmark verified by greedy decoding comparison. Restored models produce the exact same tokens. No approximation.

// Quick Start

Three lines to 17×

Works with any model vLLM supports. Single GPU or multi-GPU.

import thaw_vllm from vllm import LLM llm = LLM(model="meta-llama/Meta-Llama-3-70B-Instruct", tensor_parallel_size=2) # Freeze: each GPU saves its shard thaw_vllm.freeze_model_tp(llm, "/snapshots/llama-70b.thaw")
import thaw_vllm # Restore: 17x faster than normal cold start llm = thaw_vllm.load( "meta-llama/Meta-Llama-3-70B-Instruct", "/snapshots/llama-70b.thaw", tensor_parallel_size=2 ) # Ready. Same model, same outputs, 17x faster.
import thaw_vllm # registers the loader from vllm import LLM llm = LLM( model="meta-llama/Meta-Llama-3-70B-Instruct", load_format="thaw", tensor_parallel_size=2, model_loader_extra_config={"snapshot": "/snapshots/llama-70b.thaw"} )
